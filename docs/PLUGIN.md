## 插件接口说明

所有插件必须继承 `vector_studio.plugin_interface.Plugin` 基类，并至少定义 `name` 类属性。

```python
from vector_studio.plugin_interface import Plugin

class MyPlugin(Plugin):
    name = "my_plugin"          # 必填，唯一标识
    version = "1.0.0"         # 可选
    description = "..."         # 可选
    author = "Your Name"      # 可选
```

### 插件生命周期

1. **发现（Discover）**：`PluginManager` 扫描以下目录查找 `.py` 文件：
   - 内置插件目录：`src/vector_studio/builtin_plugins/`
   - 用户插件目录：`~/.bitmap_vector_studio/plugins/`
   - 项目插件目录：`./plugins/`（当前工作目录）
   - 额外目录：通过 `plugin_dirs` 配置或 `PluginManager` 构造函数传入

2. **注册（Register）**：发现的插件类被注册到管理器中，默认自动启用。

3. **执行（Execute）**：在转换流程的对应 Hook 点，管理器调用所有已启用且实现了该 Hook 的插件实例。

### Plugin 基类定义

```python
class Plugin:
    name: str = ""
    version: str = ""
    description: str = ""
    author: str = ""

    def preprocess(self, image: Image.Image, options: dict[str, Any]) -> Image.Image:
        """预处理 Hook：在矢量化之前修改输入图片。"""
        return image

    def postprocess(self, svg_path: Path, options: dict[str, Any]) -> Path:
        """后处理 Hook：在 SVG 优化后修改输出文件。"""
        return svg_path

    def on_convert_complete(self, result: TraceResult, options: dict[str, Any]) -> None:
        """完成 Hook：转换结束后执行副作用操作。"""
        pass
```

## 最小插件示例

创建一个 `hello_plugin.py`：

```python
from pathlib import Path
from typing import Any

from vector_studio.plugin_interface import Plugin


class HelloPlugin(Plugin):
    """A minimal hello-world plugin that logs after conversion."""

    name = "hello"
    version = "1.0.0"
    description = "Prints a message when conversion completes."
    author = "Your Name"

    def on_convert_complete(self, result, options: dict[str, Any]) -> None:
        print(f"[HelloPlugin] ✅ Conversion finished: {result.svg_path}")
        print(f"[HelloPlugin]    Duration: {result.duration:.2f}s")
```

### 安装并测试

```bash
# 方法 1：复制到用户插件目录
mkdir -p ~/.bitmap_vector_studio/plugins
cp hello_plugin.py ~/.bitmap_vector_studio/plugins/

# 方法 2：使用 CLI 安装
vector-studio plugin install ./hello_plugin.py

# 启用插件
vector-studio plugin enable hello

# 使用插件进行转换
vector-studio trace input.png --plugin hello --preset poster
```

## 预处理/后处理/完成钩子详解

### `preprocess(image, options)` — 预处理

在图片进入 VTracer 矢量化引擎**之前**调用。可用于：
- 自定义图像滤波
- 添加边框或裁剪
- 颜色空间转换
- 水印预处理（在光栅阶段添加）

**参数：**
- `image`：`PIL.Image.Image` 实例
- `options`：字典，包含当前转换的所有参数和选项

**返回值：** 修改后的 `PIL.Image.Image` 实例

**示例 — 自动裁剪透明边：**

```python
from PIL import Image
from vector_studio.plugin_interface import Plugin

class AutoCropPlugin(Plugin):
    name = "auto_crop"
    version = "1.0.0"
    description = "Automatically crop transparent borders before tracing."

    def preprocess(self, image: Image.Image, options: dict) -> Image.Image:
        if image.mode in ("RGBA", "P"):
            bbox = image.getbbox()
            if bbox:
                return image.crop(bbox)
        return image
```

---

### `postprocess(svg_path, options)` — 后处理

在 SVG 生成并经过基础优化**之后**调用。可用于：
- 注入自定义 SVG 元素（水印、元数据）
- 修改 viewBox 或尺寸
- 添加 CSS 样式
- 调用外部工具进一步处理 SVG

**参数：**
- `svg_path`：`pathlib.Path`，指向生成的 SVG 文件
- `options`：字典，包含当前转换的所有参数和选项

**返回值：** `pathlib.Path`，指向（可能修改后的）SVG 文件

**示例 — 添加元数据：**

```python
from pathlib import Path
from vector_studio.plugin_interface import Plugin

class MetadataPlugin(Plugin):
    name = "metadata"
    version = "1.0.0"
    description = "Add custom metadata to generated SVGs."

    def postprocess(self, svg_path: Path, options: dict) -> Path:
        content = svg_path.read_text(encoding="utf-8")
        metadata = f'<!-- Generated by Bitmap Vector Studio via {self.name} plugin -->\n'
        if "<svg" in content:
            content = content.replace("<svg", metadata + "<svg", 1)
            svg_path.write_text(content, encoding="utf-8")
        return svg_path
```

---

### `on_convert_complete(result, options)` — 完成回调

在**整个转换流程结束后**调用，无论成功或失败（失败时 `result` 可能为 None）。可用于：
- 发送通知（邮件、Slack、Webhook）
- 记录日志到外部系统
- 移动文件到指定目录
- 触发后续工作流

**参数：**
- `result`：`TraceResult` 实例，包含 `svg_path`、`duration`、`stats` 等属性
- `options`：字典，包含当前转换的所有参数和选项

**返回值：** 无

**示例 — Webhook 通知：**

```python
import urllib.request
import json
from vector_studio.plugin_interface import Plugin

class WebhookPlugin(Plugin):
    name = "webhook"
    version = "1.0.0"
    description = "Send a webhook notification when conversion completes."

    def on_convert_complete(self, result, options: dict) -> None:
        url = options.get("webhook_url")
        if not url:
            return
        payload = json.dumps({
            "svg_path": str(result.svg_path),
            "duration": result.duration,
            "preset": options.get("preset"),
        }).encode()
        req = urllib.request.Request(url, data=payload, headers={
            "Content-Type": "application/json"
        })
        try:
            with urllib.request.urlopen(req, timeout=5):
                pass
        except Exception as exc:
            print(f"[WebhookPlugin] Failed: {exc}")
```

## 内置插件参考

### `watermark` — SVG 文字水印

在生成的 SVG 右下角添加文字水印。

**通过 options 控制：**

| 选项键 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `watermark_text` | string | `"Generated by Bitmap Vector Studio"` | 水印文字 |
| `watermark_color` | string | `"#999999"` | 文字颜色 |
| `watermark_size` | int | `12` | 字体大小 |

**CLI 使用：**

```bash
vector-studio trace input.png --plugin watermark --preset poster
```

**Python API 使用：**

```python
from vector_studio.plugins import PluginManager
from vector_studio.tracer import trace_image

pm = PluginManager()
pm.discover_plugins()
pm.enable_plugin("watermark")

# 在 options 中传入水印参数
options = {"watermark_text": "© 2026 My Company", "watermark_color": "#666666"}
# trace_image 内部会自动调用插件 hooks
```

---

### `resize` — SVG 尺寸缩放

按比例缩放 SVG 的 `viewBox`、`width` 和 `height`。

**通过 options 控制：**

| 选项键 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `resize_scale` | float | `None` | 缩放比例，如 `2.0` 表示放大一倍，`0.5` 表示缩小一半 |

**CLI 使用：**

```bash
# 目前 CLI 暂不支持直接传递插件参数，推荐通过 Python API 使用
```

**Python API 使用：**

```python
from vector_studio.plugins import PluginManager

pm = PluginManager()
pm.discover_plugins()
pm.enable_plugin("resize")

options = {"resize_scale": 2.0}
# 插件会在 postprocess 阶段缩放 SVG
```

## 插件安装和调试

### 目录结构

```text
~/.bitmap_vector_studio/
└── plugins/
    ├── hello_plugin.py
    ├── watermark_custom.py
    └── my_company/
        └── __init__.py   # 子目录中的插件也会被扫描
```

### CLI 管理

```bash
# 列出所有插件（内置 + 用户 + 项目）
vector-studio plugin list

# 启用插件
vector-studio plugin enable watermark

# 禁用插件
vector-studio plugin disable resize

# 安装插件文件
vector-studio plugin install ./my_plugin.py

# 临时启用（仅当前命令）
vector-studio trace input.png --plugin watermark --plugin hello
```

### Python API 管理

```python
from vector_studio.plugins import PluginManager
from vector_studio.plugin_interface import Plugin

# 创建管理器并发现插件
pm = PluginManager()
pm.discover_plugins()

# 查看所有插件
for info in pm.list_plugins():
    print(f"{info['name']} v{info['version']} - {info['description']} "
          f"[enabled={info['enabled']}, hooks={info['hooks']}]")

# 手动注册插件类
class MyPlugin(Plugin):
    name = "my_plugin"
    version = "1.0.0"

pm.register_plugin(MyPlugin)

# 启用/禁用
pm.enable_plugin("my_plugin")
pm.disable_plugin("watermark")

# 执行预处理链
from PIL import Image
img = Image.open("input.png")
processed = pm.run_preprocess(img, {"preset": "logo"})
```

### 调试技巧

1. **查看日志**：插件加载失败或 Hook 执行异常时，管理器会记录 `warning` 级别日志，不会中断主流程。
   ```python
   import logging
   logging.basicConfig(level=logging.WARNING)
   ```

2. **验证插件被发现**：
   ```bash
   vector-studio plugin list
   ```
   如果插件未出现，检查：
   - 文件是否以 `.py` 结尾
   - 文件名是否以下划线开头（`_` 开头的文件会被忽略）
   - 插件类是否正确定义了 `name` 属性
   - 插件类是否继承自 `Plugin`

3. **单元测试插件**：
   ```python
   from vector_studio.plugins import PluginManager

   def test_my_plugin():
       pm = PluginManager()
       pm.register_plugin(MyPlugin)
       pm.enable_plugin("my_plugin")
       
       # 测试 preprocess
       img = Image.new("RGB", (100, 100))
       result = pm.run_preprocess(img, {})
       assert result.size == (100, 100)
   ```

## 高级示例

### 组合多个插件

```python
from vector_studio.plugins import PluginManager

pm = PluginManager()
pm.discover_plugins()

# 同时启用水印和自定义后处理
pm.enable_plugin("watermark")
pm.enable_plugin("my_custom")

options = {
    "watermark_text": "© My Brand",
    "watermark_color": "#333333",
    "custom_option": True,
}

# 在转换流程中，管理器会自动按顺序调用所有启用的插件
```

### 插件配置持久化

通过 `Config` 将常用插件配置保存到配置文件：

```python
from vector_studio.config import Config

config = Config.load()
config.enabled_plugins = ["watermark", "auto_crop"]
config.save()
```

CLI 会自动读取 `enabled_plugins` 并在转换时启用对应插件。

## 最佳实践

1. **保持幂等**：`preprocess` 和 `postprocess` 应尽可能幂等，避免重复执行导致意外结果。
2. **优雅降级**：Hook 中捕获异常并记录日志，不要让插件错误阻断主转换流程。
3. **最小侵入**：只修改你关心的部分，其他内容原样返回。
4. **命名规范**：插件 `name` 使用小写字母和下划线，避免与内置插件冲突。
5. **版本管理**：为插件定义 `version` 和 `author`，便于后续维护。
