## 安装指南

### Windows（MSI / NSIS）

**MSI 安装包（推荐）**
1. 从 [GitHub Releases](https://github.com/jammyfu/bitmap-vector-studio/releases/latest) 下载 `bitmap-vector-studio_3.0.0_x64_en-US.msi`
2. 双击运行安装向导
3. 按提示完成安装（支持自定义安装路径）
4. 安装完成后从开始菜单启动「Bitmap Vector Studio」

**NSIS 安装包**
1. 下载 `bitmap-vector-studio_3.0.0_x64-setup.exe`
2. 双击运行，支持便携模式安装（无需管理员权限）

> **系统要求**：Windows 10 版本 1809 或更高，64 位系统，Python 3.9+（首次启动时自动检测）

### macOS（DMG / App）

**DMG 镜像（推荐）**
1. 从 [GitHub Releases](https://github.com/jammyfu/bitmap-vector-studio/releases/latest) 下载 `bitmap-vector-studio_3.0.0_x64.dmg`（Intel）或 `bitmap-vector-studio_3.0.0_aarch64.dmg`（Apple Silicon）
2. 双击挂载 DMG 镜像
3. 将「Bitmap Vector Studio」应用拖入「应用程序」文件夹
4. 首次启动时若提示「无法打开」，前往「系统设置 → 隐私与安全性」点击「仍要打开」

**App 压缩包**
1. 下载 `.app.tar.gz` 文件
2. 解压后将 `.app` 拖入「应用程序」文件夹

> **系统要求**：macOS 10.13 或更高，Intel 或 Apple Silicon，Python 3.9+（首次启动时自动检测）

### Linux（AppImage / DEB / RPM）

**AppImage（推荐，通用）**
1. 从 [GitHub Releases](https://github.com/jammyfu/bitmap-vector-studio/releases/latest) 下载 `bitmap-vector-studio_3.0.0_amd64.AppImage`
2. 赋予执行权限：
   ```bash
   chmod +x bitmap-vector-studio_3.0.0_amd64.AppImage
   ```
3. 直接运行：
   ```bash
   ./bitmap-vector-studio_3.0.0_amd64.AppImage
   ```
4. 可选：使用 [AppImageLauncher](https://github.com/TheAssassin/AppImageLauncher) 集成到系统菜单

**DEB 包（Debian / Ubuntu）**
```bash
sudo dpkg -i bitmap-vector-studio_3.0.0_amd64.deb
sudo apt-get install -f  # 自动解决依赖
```

**RPM 包（Fedora / openSUSE）**
```bash
sudo rpm -i bitmap-vector-studio_3.0.0_x86_64.rpm
```

> **系统要求**：Linux 内核 3.10+，glibc 2.17+，Python 3.9+ 和 libcairo2（首次启动时自动检测）

## 首次启动配置

### 1. Python 环境检测

桌面应用依赖本地 Python 环境运行矢量化引擎。首次启动时会自动检测：

- ✅ **已检测到 `vector-studio` 命令**：直接启动
- ⚠️ **检测到 Python 但未安装包**：弹出提示，一键安装 `pip install vector-studio`
- ❌ **未检测到 Python**：引导下载 Python 3.9+ 并提供安装指南

### 2. 工作区设置

首次启动后建议配置：

- **默认输出目录**：设置 SVG 文件的默认保存位置
- **默认预设**：选择最常用的转换预设（推荐 `poster` 或 `logo`）
- **自动检查更新**：建议开启，启动时自动检测新版本
- **主题**：跟随系统 / 浅色 / 深色

配置保存在 `~/.bitmap_vector_studio/config.yaml`，与 CLI 和 Streamlit GUI 共享。

## 界面介绍

桌面应用采用「单栏聚焦 + 底部控制带」布局，将视觉焦点集中在中央画布：

```
┌─────────────────────────────────────────────────────────────┐
│  Logo    🔍 Command Palette    设置  主题                    │  ← 顶部栏 (56px)
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    中央预览画布                               │  ← MainCanvas
│                                                             │
│           并排对比 / 叠加对比滑块                             │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  [预设]  [核心1] [核心2] [核心3] [核心4]  [开始转换]         │  ← 底部控制带 (72px)
├─────────────────────────────────────────────────────────────┤
│  ▼ 文件队列 (可折叠)                                         │  ← QueueBar
│  📄 img1  📄 img2  📄 img3 ...                              │
└─────────────────────────────────────────────────────────────┘
```

### 顶部栏（TopBar）

- **Logo**：点击返回首页/清空状态
- **Command Palette**：`Cmd/Ctrl+K` 打开，统一搜索所有功能、预设、文件和设置
- **设置/主题**：快速切换浅色/深色/跟随系统主题

### 中央画布（MainCanvas）

- **并排对比**：左侧原图，右侧 SVG 结果，同步缩放和拖拽
- **叠加对比**：拖动滑块查看原图与矢量化结果的差异
- **拖拽上传**：直接将图片拖拽到画布区域即可加入队列并自动分析
- **缩放控制**：鼠标滚轮缩放，双击重置，支持 25% - 400%
- **空状态引导**：无文件时显示拖拽提示和最近文件快捷入口

### 底部控制带（ControlBar）

- **4 个核心参数**：颜色模式、曲线模式、颜色精度、滤斑点
- **预设选择器**：快速切换内置/用户预设，显示智能推荐标签
- **转换按钮**：主操作按钮，暖橙色强调，显示当前状态
- **高级参数抽屉**：点击「高级」从底部滑出，默认折叠

### 文件队列（QueueBar）

- 可折叠底部栏，显示当前待处理文件
- 支持多文件拖拽（1-50 个文件）
- 显示转换进度和状态（等待中/转换中/已完成/失败）
- 支持删除、重新排序、一键导出全部

## 新布局架构

v3.0 将原有的三栏布局（左队列/中画布/右参数）重构为「单栏聚焦 + 底部控制带」架构，核心变化：

| 旧布局 (v2.0) | 新布局 (v3.0) | 设计意图 |
|---|---|---|
| 三栏并排，视觉分散 | 单栏聚焦，画布居中 | 减少视觉跳跃，专注创作 |
| 右侧参数面板常驻 | 底部控制带，仅4个核心参数 | 降低认知负荷 |
| 高级参数全部展开 | 默认折叠在底部抽屉 | 渐进式披露 |
| 手动选择预设 | 智能默认自动推荐 | 减少操作步骤 |
| 功能分散在各面板 | Command Palette 统一入口 | 统一搜索，快速触达 |

### 组件清单

v3.0 桌面端由 8 个核心组件构成：

| 组件 | 职责 | 文件 |
|---|---|---|
| **TopBar** | 全局导航、Command Palette 入口、主题切换 | `TopBar.tsx` |
| **MainCanvas** | 图片预览、对比模式、拖拽上传、缩放控制 | `MainCanvas.tsx` |
| **CoreParams** | 4 个核心参数控件（颜色模式/曲线模式/颜色精度/滤斑点） | `CoreParams.tsx` |
| **SmartRecommend** | 智能预设推荐标签、置信度显示、一键应用 | `SmartRecommend.tsx` |
| **AdvancedDrawer** | 底部滑出抽屉，容纳所有高级参数和选项 | `AdvancedDrawer.tsx` |
| **ControlBar** | 底部控制带容器，编排预设+参数+转换按钮 | `ControlBar.tsx` |
| **QueueBar** | 可折叠文件队列，显示任务状态和进度 | `QueueBar.tsx` |
| **CommandPalette** | 全局命令面板，搜索功能/预设/文件/设置 | `CommandPalette.tsx` |

### 状态管理

v3.0 使用 Zustand 5 个 Store 替代原有的 40+ useState：

| Store | 管理内容 |
|---|---|
| `useAppStore` | 全局应用状态（主题、加载态、错误） |
| `useFileStore` | 文件队列、当前选中文件、拖拽状态 |
| `useParamStore` | 所有转换参数、预设选择、高级参数 |
| `useUIStore` | UI 布局状态（抽屉开关、面板显隐、缩放比例） |
| `useProcessStore` | 转换进度、任务状态、历史记录 |

## Command Palette

Command Palette 是 v3.0 的核心交互创新，通过 `Cmd/Ctrl+K` 呼出，提供统一的全局搜索入口。

### 打开方式

- 快捷键：`Cmd+K`（macOS）/ `Ctrl+K`（Windows/Linux）
- 点击顶部栏的 🔍 搜索图标
- 从菜单栏「视图 → Command Palette」

### 搜索范围

| 类型 | 示例 | 操作 |
|---|---|---|
| **功能命令** | "转换"、"导出 PDF"、"打开设置" | 直接执行 |
| **预设** | "logo"、"poster"、"bw" | 切换当前预设 |
| **文件** | "img1.png"、最近打开的文件 | 选中并加载 |
| **设置项** | "主题"、"GPU 加速"、"语言" | 跳转到对应设置 |
| **快捷键** | "撤销"、"全屏" | 显示并执行 |

### 使用示例

1. 按 `Cmd/Ctrl+K` 打开面板
2. 输入 "logo" → 显示 logo 预设，回车直接应用
3. 输入 "导出 pdf" → 执行当前文件的 PDF 导出
4. 输入 "深色" → 切换至深色主题
5. 按 `Esc` 或点击遮罩关闭面板

### 设计特点

- 顶部居中悬浮，占据视口宽度 60%（最大 720px）
- 背景遮罩，突出面板层级
- 实时过滤，无延迟响应
- 最近使用命令置顶显示
- 支持键盘上下导航和回车确认

## 智能默认系统

v3.0 引入智能默认系统，目标是让用户上传图片后，无需任何手动操作即可获得最佳转换结果。

### 工作流程

```
用户拖拽图片 → 自动分析特征 → 推荐最佳预设 → [高置信度] 自动应用
                                      ↓
                              [低置信度] 显示推荐标签，等待用户确认
```

### 分析维度

| 维度 | 说明 | 影响 |
|---|---|---|
| **颜色数量** | 统计图片主色调数量 | 决定 `color` 或 `binary` 模式 |
| **边缘密度** | 检测边缘像素占比 | 影响 `filter_speckle` 和 `corner_threshold` |
| **宽高比** | 图片长宽比例 | 影响 `max_input_side` 和裁剪策略 |
| **对比度** | 明暗区域差异程度 | 影响 `layer_difference` |
| **噪点水平** | 高频噪声估计 | 决定是否启用 `denoise` |

### 置信度规则

- **高置信度（≥85%）**：自动应用推荐预设，显示 Toast 通知「已自动应用 Logo 预设」
- **中置信度（60%-84%）**：在控制带显示推荐标签，用户点击一键应用
- **低置信度（<60%）**：显示多个候选预设，由用户选择

### 手动覆盖

智能默认始终可被手动覆盖：
- 点击控制带的预设选择器手动切换
- 调整任何核心参数后，智能推荐标签变为「自定义」
- 在设置中可关闭「自动应用高置信度推荐」

## 响应式行为

桌面应用支持三种显示模式，自动根据窗口尺寸调整布局：

### 桌面模式（≥1024px）

- 完整布局，所有组件展开
- 底部控制带显示 4 个核心参数 + 预设 + 转换按钮
- 文件队列默认展开
- Command Palette 宽度 60%（最大 720px）

### 紧凑模式（768px - 1023px）

- 控制带参数收拢为图标 + 下拉选择器
- 文件队列默认折叠，显示为底部标签
- 画布区域最大化
- 高级参数抽屉高度调整为 80%

### 移动端/小窗口模式（<768px）

- 控制带变为底部固定窄条（48px 高）
- 核心参数收纳入底部抽屉
- 文件队列变为侧边滑出面板
- 画布全高显示，对比控件悬浮叠加
- Command Palette 全屏显示

### 手动调整

在「设置 → 外观」中可：
- 强制锁定为桌面模式（即使窗口较小）
- 调整控制带密度（标准/紧凑）
- 设置画布最小边距

## 文件拖拽使用

### 拖拽到画布

v3.0 推荐将文件直接拖拽到中央画布区域：

1. 在文件管理器中选择一张或多张图片
2. 拖拽到应用窗口中央画布区域
3. 文件自动加入队列，显示缩略图和文件名
4. 自动触发智能分析并显示预设推荐
5. 点击文件即可在预览画布中查看

### 支持的格式

| 格式 | 扩展名 | 说明 |
|---|---|---|
| PNG | `.png` | 推荐，支持透明通道 |
| JPEG | `.jpg`, `.jpeg` | 常见照片格式 |
| WebP | `.webp` | 现代网页格式 |
| BMP | `.bmp` | Windows 位图 |
| TIFF | `.tif`, `.tiff` | 印刷级格式 |

> 不支持 GIF、SVG、RAW 等格式，拖拽时会自动过滤并提示。

### 拖拽技巧

- **多文件拖拽**：支持同时拖拽 1 - 50 个文件，超过限制会提示分批处理
- **文件夹拖拽**：暂不支持直接拖拽文件夹，请展开后选择图片文件
- **从浏览器拖拽**：支持从网页直接拖拽图片到应用窗口
- **拖拽排序**：在队列列表中拖拽文件可调整处理顺序

## 队列管理

### 添加任务

- **方式一**：文件 → 打开（`Ctrl+O`），选择图片后自动加入队列
- **方式二**：直接将图片文件拖拽到画布区域（v3.0 推荐）
- **方式三**：点击队列底部的「+」按钮，通过文件对话框选择

### 任务状态

| 状态 | 图标 | 说明 |
|---|---|---|
| 等待中 | ⏳ | 已加入队列，等待处理 |
| 分析中 | 🔍 | 正在进行智能特征分析 |
| 转换中 | 🔄 | 正在进行矢量化 |
| 预览中 | 👁️ | 正在生成实时预览 |
| 已完成 | ✅ | 转换成功，可导出 SVG |
| 失败 | ❌ | 转换失败，点击查看错误信息 |

### 批量操作

- **开始全部**：点击「开始批量转换」或 `Ctrl+Shift+T`
- **暂停/继续**：点击暂停按钮暂停队列，再次点击继续
- **清空队列**：点击「清空」按钮，移除所有等待中的任务
- **导出全部**：所有任务完成后，点击「导出全部」选择输出目录
- **删除已完成**：一键清理已完成的任务，保留失败的以便重试

### 队列设置

在「设置」→「队列」中可配置：
- **并发数**：同时处理的任务数（默认 2，建议根据 CPU 核心数调整 1-4）
- **失败重试**：自动重试失败任务的次数（默认 1）
- **自动开始**：添加文件后是否自动开始转换（默认关闭）
- **完成通知**：任务完成后显示系统通知（默认开启）

## 自动更新

### 更新检查

桌面应用内置 Tauri updater，支持以下更新方式：

- **启动时检查**：应用启动后自动检测 GitHub Releases 是否有新版本
- **手动检查**：帮助菜单 → 检查更新
- **后台检查**：每 24 小时自动检查一次（仅在应用运行期间）

### 更新流程

1. 检测到新版本时，状态栏显示「🎉 新版本可用」提示
2. 点击提示或前往「帮助 → 检查更新」查看更新日志
3. 点击「下载并安装」自动下载更新包
4. 下载完成后提示重启应用以完成安装
5. 重启后自动应用更新

### 更新通道

| 通道 | 说明 | 适用场景 |
|---|---|---|
| `stable` | 稳定版本，经过完整测试 | 生产环境、日常使用 |
| `beta` | 测试版本，包含新功能预览 | 愿意尝试新功能的用户 |

在「设置 → 更新」中可切换更新通道（默认 `stable`）。

### 离线环境

若处于无网络环境或内网部署：
1. 在联网设备下载最新版本的安装包
2. 手动安装覆盖旧版本

## 性能设置

v1.1 在「设置 → 性能」标签页提供以下配置：

### GPU 加速

| 选项 | 说明 | 默认值 |
|---|---|---|
| **启用 GPU 加速** | 自动检测并使用 CUDA / Metal / OpenCL 加速矢量化 | 开启 |
| **GPU 降级策略** | GPU 初始化失败时自动切换到 CPU | 开启 |

> **注意**：GPU 加速需要对应的驱动程序。NVIDIA 需 CUDA Toolkit，macOS 自动使用 Metal，Linux 可尝试 OpenCL。

### 大文件流式处理

| 选项 | 说明 | 默认值 |
|---|---|---|
| **流式处理阈值** | 超过此大小的图片自动启用分块读取（MB） | 100 |
| **分块大小** | 每次读取的内存块大小（MB） | 32 |

> 处理超大 TIFF/PNG 时，流式处理可避免内存溢出。建议内存 8GB 以下用户将阈值设为 50MB。

### 内存监控

| 选项 | 说明 | 默认值 |
|---|---|---|
| **内存限制** | 当内存占用超过此百分比时触发保护策略 | 80% |
| **自动降频** | 内存紧张时自动降低并发数和预览分辨率 | 开启 |

## 工作区管理

v1.1 引入工作区自动保存机制，防止意外崩溃导致进度丢失。

### 自动保存

在「设置 → 工作区」中配置：

| 选项 | 说明 | 默认值 |
|---|---|---|
| **自动保存间隔** | 每隔多少秒自动保存一次工作区状态 | 60 |
| **崩溃恢复** | 异常退出后重启时提示恢复工作区 | 开启 |
| **最大工作区数** | 保留的最近工作区快照数量 | 10 |

### 工作区内容

自动保存的工作区包含：
- 当前打开的文件队列（含文件路径和状态）
- 参数面板的所有设置
- 预览画布的缩放和模式状态
- 插件启用状态
- 历史面板数据
- **v3.0 新增**：Command Palette 最近使用记录、抽屉展开状态

### 手动管理工作区

```bash
# 保存当前工作区
vector-studio workspace save project_a

# 加载工作区
vector-studio workspace load project_a

# 列出所有工作区
vector-studio workspace list

# 删除工作区
vector-studio workspace delete project_a
```

## 断点续传

v1.1 的批量任务支持断点续传，异常中断后可恢复未完成的任务。

### 自动检查点

批量转换时，`CheckpointManager` 自动按以下策略保存进度：
- 每完成 5 个文件保存一次检查点
- 异常中断时立即保存当前进度
- 检查点保留 7 天，过期自动清理

### 恢复任务

**桌面端**：
1. 重启应用后，若检测到未完成的检查点，队列面板显示「🔄 可恢复任务」提示
2. 点击「恢复」按钮，自动加载未完成的文件并继续转换
3. 已完成的文件不会重复处理

**CLI**：
```bash
# 列出可恢复的检查点
vector-studio resume list

# 恢复指定检查点
vector-studio resume checkpoint-id

# 清除已完成的检查点
vector-studio resume clear
```

### 检查点文件位置

检查点保存在 `~/.bitmap_vector_studio/checkpoints/` 目录，每个检查点为一个 JSON 文件，包含：
- 任务队列状态
- 已完成/失败/待处理的文件列表
- 转换参数和选项

## OCR 多语言设置

v1.1 增强 OCR 功能，支持 10 种语言和竖排文字检测。

### 语言配置

在「设置 → OCR」中配置：

| 选项 | 说明 | 默认值 |
|---|---|---|
| **默认语言** | OCR 识别时优先使用的语言包 | `chi_sim+eng` |
| **竖排检测** | 自动检测竖排排版文字 | 开启 |
| **Tesseract 路径** | 自定义 Tesseract 可执行文件路径 | 自动检测 |

### 支持的语言

| 语言 | 代码 | 说明 |
|---|---|---|
| 简体中文 | `chi_sim` | 常用中文识别 |
| 繁体中文 | `chi_tra` | 台湾、香港繁体 |
| 日文 | `jpn` | 含平假名、片假名、汉字 |
| 韩文 | `kor` | 韩文识别 |
| 阿拉伯文 | `ara` | 从右至左排版 |
| 俄文 | `rus` | 西里尔字母 |
| 德文 | `deu` | 德文识别 |
| 法文 | `fra` | 法文识别 |
| 西班牙文 | `spa` | 西班牙文识别 |
| 英文 | `eng` | 英文识别 |

### 多语言组合

支持用 `+` 连接多个语言包：
- `chi_sim+eng`：中文 + 英文混合（最常见）
- `jpn+eng`：日文 + 英文
- `kor+chi_sim`：韩文 + 中文

### 竖排文字

开启「竖排检测」后，OCR 模块会自动：
1. 检测文字区域是否为竖排排版
2. 对竖排区域使用竖排识别模式
3. 在 SVG 中生成竖排 `<text>` 元素（通过 `writing-mode: vertical-rl`）

适用场景：日文传统书籍、中文古籍、竖排海报。

## 故障排查

### 启动问题

#### 应用无法启动 / 闪退

1. **检查 Python 环境**：确保 `python3` 或 `python` 在系统 PATH 中
2. **检查依赖安装**：运行 `pip install vector-studio` 确保 Python 包已安装
3. **查看日志**：
   - Windows：`%APPDATA%\Bitmap Vector Studio\logs\`
   - macOS：`~/Library/Logs/Bitmap Vector Studio/`
   - Linux：`~/.config/Bitmap Vector Studio/logs/`
4. **重置配置**：删除 `~/.bitmap_vector_studio/config.yaml` 后重启

#### 首次启动卡在「检测环境」

- 手动安装 Python 包：`pip install vector-studio`
- 若使用虚拟环境，确保虚拟环境的 `bin` 目录在 PATH 中
- 检查网络连接（首次启动可能需要下载依赖）

### 转换问题

#### 转换失败 / 无响应

| 现象 | 可能原因 | 解决方案 |
|---|---|---|
| 所有文件转换失败 | Python 后端未正确启动 | 检查日志，重新安装 `vector-studio` |
| 特定文件失败 | 文件格式不支持或损坏 | 尝试用其他软件打开并重新保存 |
| 大文件无响应 | 内存不足 | 在高级参数中设置「限制输入边长」为 2400 |
| 预览不更新 | 实时预览被暂停 | 按 `Space` 或点击「恢复预览」 |

#### 输出 SVG 质量不佳

1. **选择合适的预设**：Logo 用 `logo`，照片用 `photo`，插画用 `poster`
2. **调整颜色精度**：提高 `color_precision`（1-8，越高颜色越丰富）
3. **使用 AI 简化**：对复杂照片先启用「AI 语义简化」再转换
4. **参数搜索**：在 CLI 中使用 `vector-studio search image.png` 自动找最优参数

### 界面问题

#### 预览画布黑屏 / 不显示

- 检查文件是否成功加载（队列中应显示缩略图）
- 尝试切换预览模式（并排 / 叠加）
- 重置缩放：`Ctrl+0`
- 检查显卡驱动（WebView2 / WebKit 需要 GPU 加速）

#### 字体显示异常

- Windows：确保系统已安装「Microsoft YaHei」或「Segoe UI」
- Linux：安装 Noto 字体包：`sudo apt install fonts-noto`
- 在「设置 → 外观」中调整界面字体

#### 控制带或抽屉显示异常

- 尝试切换显示模式：「视图 → 强制桌面模式」
- 重置窗口大小至 1280×800 以上
- 检查是否启用了系统级缩放（建议 100% 或 125%）

### 性能问题

#### 应用卡顿 / 内存占用高

1. **减少并发数**：设置 → 队列 → 并发数改为 1
2. **关闭实时预览**：处理大文件时暂停实时预览
3. **限制输入边长**：高级参数 → 预处理 → 最大输入边长设为 1600-2400
4. **清理历史**：历史面板 → 清空历史（保留最近 20 条即可）

#### 启动速度慢

- 首次启动需要初始化 Python 环境，后续启动会更快
- 关闭「启动时检查更新」可略微提升启动速度
- 在 SSD 上运行应用，避免机械硬盘
