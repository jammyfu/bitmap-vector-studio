## 安装指南

### Windows（MSI / NSIS）

**MSI 安装包（推荐）**
1. 从 [GitHub Releases](https://github.com/jammyfu/bitmap-vector-studio/releases/latest) 下载 `bitmap-vector-studio_1.2.0_x64_en-US.msi`
2. 双击运行安装向导
3. 按提示完成安装（支持自定义安装路径）
4. 安装完成后从开始菜单启动「Bitmap Vector Studio」

**NSIS 安装包**
1. 下载 `bitmap-vector-studio_1.2.0_x64-setup.exe`
2. 双击运行，支持便携模式安装（无需管理员权限）

> **系统要求**：Windows 10 版本 1809 或更高，64 位系统，Python 3.9+（首次启动时自动检测）

### macOS（DMG / App）

**DMG 镜像（推荐）**
1. 从 [GitHub Releases](https://github.com/jammyfu/bitmap-vector-studio/releases/latest) 下载 `bitmap-vector-studio_1.2.0_x64.dmg`（Intel）或 `bitmap-vector-studio_1.2.0_aarch64.dmg`（Apple Silicon）
2. 双击挂载 DMG 镜像
3. 将「Bitmap Vector Studio」应用拖入「应用程序」文件夹
4. 首次启动时若提示「无法打开」，前往「系统设置 → 隐私与安全性」点击「仍要打开」

**App 压缩包**
1. 下载 `.app.tar.gz` 文件
2. 解压后将 `.app` 拖入「应用程序」文件夹

> **系统要求**：macOS 10.13 或更高，Intel 或 Apple Silicon，Python 3.9+（首次启动时自动检测）

### Linux（AppImage / DEB / RPM）

**AppImage（推荐，通用）**
1. 从 [GitHub Releases](https://github.com/jammyfu/bitmap-vector-studio/releases/latest) 下载 `bitmap-vector-studio_1.2.0_amd64.AppImage`
2. 赋予执行权限：
   ```bash
   chmod +x bitmap-vector-studio_1.2.0_amd64.AppImage
   ```
3. 直接运行：
   ```bash
   ./bitmap-vector-studio_1.2.0_amd64.AppImage
   ```
4. 可选：使用 [AppImageLauncher](https://github.com/TheAssassin/AppImageLauncher) 集成到系统菜单

**DEB 包（Debian / Ubuntu）**
```bash
sudo dpkg -i bitmap-vector-studio_1.2.0_amd64.deb
sudo apt-get install -f  # 自动解决依赖
```

**RPM 包（Fedora / openSUSE）**
```bash
sudo rpm -i bitmap-vector-studio_1.2.0_x86_64.rpm
```

> **系统要求**：Linux 内核 3.10+，glibc 2.17+，Python 3.9+ 和 libcairo2（首次启动时自动检测）

## 首次启动配置

### 1. Python 环境检测

桌面应用依赖本地 Python 环境运行矢量化引擎。首次启动时会自动检测：

- ✅ **已检测到 `vector-studio` 命令**：直接启动
- ⚠️ **检测到 Python 但未安装包**：弹出提示，一键安装 `pip install vector-studio`
- ❌ **未检测到 Python**：引导下载 Python 3.9+ 并提供安装指南

### 2. 工作区设置

首次启动后建议配置：

- **默认输出目录**：设置 SVG 文件的默认保存位置
- **默认预设**：选择最常用的转换预设（推荐 `poster` 或 `logo`）
- **自动检查更新**：建议开启，启动时自动检测新版本
- **主题**：跟随系统 / 浅色 / 深色

配置保存在 `~/.bitmap_vector_studio/config.yaml`，与 CLI 和 Streamlit GUI 共享。

## 界面介绍

桌面应用采用**三栏布局**，充分利用宽屏显示空间：

```
┌─────────────────────────────────────────────────────────────┐
│  文件  编辑  视图  工具  帮助                                    │  ← 菜单栏
├──────────┬──────────────────────────────┬───────────────────┤
│          │                              │                   │
│  文件队列  │      预览画布                  │    参数面板        │
│          │                              │                   │
│  📄 img1  │   ┌─────────────────────┐   │  转换预设          │
│  📄 img2  │   │                     │   │  ├── 内置预设       │
│  📄 img3  │   │   原图 / SVG 预览     │   │  └── 用户预设       │
│          │   │                     │   │                   │
│  [+ 拖拽] │   │   并排对比 / 叠加滑块   │   │  核心参数          │
│          │   │                     │   │  ├── 颜色模式       │
│          │   └─────────────────────┘   │  ├── 曲线模式       │
│          │                              │  └── ...           │
│          │   缩放: 100%  状态: 就绪      │                   │
│          │                              │  高级参数          │
├──────────┴──────────────────────────────┴───────────────────┤
│  状态栏: 就绪  |  队列: 0/0  |  版本: 1.2.0                      │
└─────────────────────────────────────────────────────────────┘
```

### 左栏：文件队列

- **拖拽区域**：将图片文件拖拽到此处加入队列
- **队列列表**：显示所有待处理文件，支持删除、重新排序
- **批量操作**：全选、清空、一键导出全部
- **进度显示**：每个文件显示转换进度和状态图标

### 中栏：预览画布

- **并排对比**：左侧原图，右侧 SVG 结果，同步缩放和拖拽
- **叠加对比**：拖动滑块查看原图与矢量化结果的差异
- **实时预览**：参数调整时 600ms 防抖后自动更新预览（可暂停）
- **缩放控制**：鼠标滚轮缩放，双击重置，支持 25% - 400%
- **选区工具**：矩形 / 圆形 / 多边形选区，用于局部重描摹

### 右栏：参数面板

- **转换预设**：内置预设 + 用户预设分组选择，保存/删除自定义预设
- **核心参数**：颜色模式、分层方式、曲线拟合、滤斑点、颜色精度
- **高级参数**：最大迭代次数、拼接阈值、路径精度
- **预处理**：降噪、限制输入边长、Posterize 颜色量化
- **智能功能**：背景透明、图像增强、预设推荐
- **AI 辅助**：实时预览开关、AI 语义简化、OCR 文字识别、局部重描摹
- **性能设置**：GPU 加速开关、流式处理阈值、内存限制（v1.1）
- **工作区管理**：自动保存间隔、崩溃恢复开关（v1.1）
- **OCR 设置**：默认语言、竖排检测开关、Tesseract 路径配置（v1.1）
- **导出选项**：压缩 SVG、导出 PDF/PNG、优化级别、质量评分
- **最近任务**：最近 5 条任务记录，一键加载参数

## 文件拖拽使用

### 拖拽到队列

1. 在文件管理器中选择一张或多张图片
2. 拖拽到应用窗口左侧的队列区域
3. 文件自动加入队列，显示缩略图和文件名
4. 点击文件即可在预览画布中查看

### 支持的格式

| 格式 | 扩展名 | 说明 |
|---|---|---|
| PNG | `.png` | 推荐，支持透明通道 |
| JPEG | `.jpg`, `.jpeg` | 常见照片格式 |
| WebP | `.webp` | 现代网页格式 |
| BMP | `.bmp` | Windows 位图 |
| TIFF | `.tif`, `.tiff` | 印刷级格式 |

> 不支持 GIF、SVG、RAW 等格式，拖拽时会自动过滤并提示。

### 拖拽技巧

- **多文件拖拽**：支持同时拖拽 1 - 50 个文件，超过限制会提示分批处理
- **文件夹拖拽**：暂不支持直接拖拽文件夹，请展开后选择图片文件
- **从浏览器拖拽**：支持从网页直接拖拽图片到应用窗口
- **拖拽排序**：在队列列表中拖拽文件可调整处理顺序

## 队列管理

### 添加任务

- **方式一**：文件 → 打开（`Ctrl+O`），选择图片后自动加入队列
- **方式二**：直接将图片文件拖拽到队列区域
- **方式三**：点击队列底部的「+」按钮，通过文件对话框选择

### 任务状态

| 状态 | 图标 | 说明 |
|---|---|---|
| 等待中 | ⏳ | 已加入队列，等待处理 |
| 转换中 | 🔄 | 正在进行矢量化 |
| 预览中 | 👁️ | 正在生成实时预览 |
| 已完成 | ✅ | 转换成功，可导出 SVG |
| 失败 | ❌ | 转换失败，点击查看错误信息 |

### 批量操作

- **开始全部**：点击「开始批量转换」或 `Ctrl+Shift+T`
- **暂停/继续**：点击暂停按钮暂停队列，再次点击继续
- **清空队列**：点击「清空」按钮，移除所有等待中的任务
- **导出全部**：所有任务完成后，点击「导出全部」选择输出目录
- **删除已完成**：一键清理已完成的任务，保留失败的以便重试

### 队列设置

在「设置」→「队列」中可配置：
- **并发数**：同时处理的任务数（默认 2，建议根据 CPU 核心数调整 1-4）
- **失败重试**：自动重试失败任务的次数（默认 1）
- **自动开始**：添加文件后是否自动开始转换（默认关闭）
- **完成通知**：任务完成后显示系统通知（默认开启）

## 自动更新

### 更新检查

桌面应用内置 Tauri updater，支持以下更新方式：

- **启动时检查**：应用启动后自动检测 GitHub Releases 是否有新版本
- **手动检查**：帮助菜单 → 检查更新
- **后台检查**：每 24 小时自动检查一次（仅在应用运行期间）

### 更新流程

1. 检测到新版本时，状态栏显示「🎉 新版本可用」提示
2. 点击提示或前往「帮助 → 检查更新」查看更新日志
3. 点击「下载并安装」自动下载更新包
4. 下载完成后提示重启应用以完成安装
5. 重启后自动应用更新

### 更新通道

| 通道 | 说明 | 适用场景 |
|---|---|---|
| `stable` | 稳定版本，经过完整测试 | 生产环境、日常使用 |
| `beta` | 测试版本，包含新功能预览 | 愿意尝试新功能的用户 |

在「设置 → 更新」中可切换更新通道（默认 `stable`）。

### 离线环境

若处于无网络环境或内网部署：
1. 在联网设备下载最新版本的安装包
2. 手动安装覆盖旧版本
3. 用户配置和预设会自动保留

## 性能设置

v1.1 在「设置 → 性能」标签页提供以下配置：

### GPU 加速

| 选项 | 说明 | 默认值 |
|---|---|---|
| **启用 GPU 加速** | 自动检测并使用 CUDA / Metal / OpenCL 加速矢量化 | 开启 |
| **GPU 降级策略** | GPU 初始化失败时自动切换到 CPU | 开启 |

> **注意**：GPU 加速需要对应的驱动程序。NVIDIA 需 CUDA Toolkit，macOS 自动使用 Metal，Linux 可尝试 OpenCL。

### 大文件流式处理

| 选项 | 说明 | 默认值 |
|---|---|---|
| **流式处理阈值** | 超过此大小的图片自动启用分块读取（MB） | 100 |
| **分块大小** | 每次读取的内存块大小（MB） | 32 |

> 处理超大 TIFF/PNG 时，流式处理可避免内存溢出。建议内存 8GB 以下用户将阈值设为 50MB。

### 内存监控

| 选项 | 说明 | 默认值 |
|---|---|---|
| **内存限制** | 当内存占用超过此百分比时触发保护策略 | 80% |
| **自动降频** | 内存紧张时自动降低并发数和预览分辨率 | 开启 |

## 工作区管理

v1.1 引入工作区自动保存机制，防止意外崩溃导致进度丢失。

### 自动保存

在「设置 → 工作区」中配置：

| 选项 | 说明 | 默认值 |
|---|---|---|
| **自动保存间隔** | 每隔多少秒自动保存一次工作区状态 | 60 |
| **崩溃恢复** | 异常退出后重启时提示恢复工作区 | 开启 |
| **最大工作区数** | 保留的最近工作区快照数量 | 10 |

### 工作区内容

自动保存的工作区包含：
- 当前打开的文件队列（含文件路径和状态）
- 参数面板的所有设置
- 预览画布的缩放和模式状态
- 插件启用状态
- 历史面板数据

### 手动管理工作区

```bash
# 保存当前工作区
vector-studio workspace save project_a

# 加载工作区
vector-studio workspace load project_a

# 列出所有工作区
vector-studio workspace list

# 删除工作区
vector-studio workspace delete project_a
```

## 断点续传

v1.1 的批量任务支持断点续传，异常中断后可恢复未完成的任务。

### 自动检查点

批量转换时，`CheckpointManager` 自动按以下策略保存进度：
- 每完成 5 个文件保存一次检查点
- 异常中断时立即保存当前进度
- 检查点保留 7 天，过期自动清理

### 恢复任务

**桌面端**：
1. 重启应用后，若检测到未完成的检查点，队列面板显示「🔄 可恢复任务」提示
2. 点击「恢复」按钮，自动加载未完成的文件并继续转换
3. 已完成的文件不会重复处理

**CLI**：
```bash
# 列出可恢复的检查点
vector-studio resume list

# 恢复指定检查点
vector-studio resume checkpoint-id

# 清除已完成的检查点
vector-studio resume clear
```

### 检查点文件位置

检查点保存在 `~/.bitmap_vector_studio/checkpoints/` 目录，每个检查点为一个 JSON 文件，包含：
- 任务队列状态
- 已完成/失败/待处理的文件列表
- 转换参数和选项

## OCR 多语言设置

v1.1 增强 OCR 功能，支持 10 种语言和竖排文字检测。

### 语言配置

在「设置 → OCR」中配置：

| 选项 | 说明 | 默认值 |
|---|---|---|
| **默认语言** | OCR 识别时优先使用的语言包 | `chi_sim+eng` |
| **竖排检测** | 自动检测竖排排版文字 | 开启 |
| **Tesseract 路径** | 自定义 Tesseract 可执行文件路径 | 自动检测 |

### 支持的语言

| 语言 | 代码 | 说明 |
|---|---|---|
| 简体中文 | `chi_sim` | 常用中文识别 |
| 繁体中文 | `chi_tra` | 台湾、香港繁体 |
| 日文 | `jpn` | 含平假名、片假名、汉字 |
| 韩文 | `kor` | 韩文识别 |
| 阿拉伯文 | `ara` | 从右至左排版 |
| 俄文 | `rus` | 西里尔字母 |
| 德文 | `deu` | 德文识别 |
| 法文 | `fra` | 法文识别 |
| 西班牙文 | `spa` | 西班牙文识别 |
| 英文 | `eng` | 英文识别 |

### 多语言组合

支持用 `+` 连接多个语言包：
- `chi_sim+eng`：中文 + 英文混合（最常见）
- `jpn+eng`：日文 + 英文
- `kor+chi_sim`：韩文 + 中文

### 竖排文字

开启「竖排检测」后，OCR 模块会自动：
1. 检测文字区域是否为竖排排版
2. 对竖排区域使用竖排识别模式
3. 在 SVG 中生成竖排 `<text>` 元素（通过 `writing-mode: vertical-rl`）

适用场景：日文传统书籍、中文古籍、竖排海报。

## 故障排查

### 启动问题

#### 应用无法启动 / 闪退

1. **检查 Python 环境**：确保 `python3` 或 `python` 在系统 PATH 中
2. **检查依赖安装**：运行 `pip install vector-studio` 确保 Python 包已安装
3. **查看日志**：
   - Windows：`%APPDATA%\Bitmap Vector Studio\logs\`
   - macOS：`~/Library/Logs/Bitmap Vector Studio/`
   - Linux：`~/.config/Bitmap Vector Studio/logs/`
4. **重置配置**：删除 `~/.bitmap_vector_studio/config.yaml` 后重启

#### 首次启动卡在「检测环境」

- 手动安装 Python 包：`pip install vector-studio`
- 若使用虚拟环境，确保虚拟环境的 `bin` 目录在 PATH 中
- 检查网络连接（首次启动可能需要下载依赖）

### 转换问题

#### 转换失败 / 无响应

| 现象 | 可能原因 | 解决方案 |
|---|---|---|
| 所有文件转换失败 | Python 后端未正确启动 | 检查日志，重新安装 `vector-studio` |
| 特定文件失败 | 文件格式不支持或损坏 | 尝试用其他软件打开并重新保存 |
| 大文件无响应 | 内存不足 | 在预处理中设置「限制输入边长」为 2400 |
| 预览不更新 | 实时预览被暂停 | 按 `Space` 或点击「恢复预览」 |

#### 输出 SVG 质量不佳

1. **选择合适的预设**：Logo 用 `logo`，照片用 `photo`，插画用 `poster`
2. **调整颜色精度**：提高 `color_precision`（1-8，越高颜色越丰富）
3. **使用 AI 简化**：对复杂照片先启用「AI 语义简化」再转换
4. **参数搜索**：在 CLI 中使用 `vector-studio search image.png` 自动找最优参数

### 界面问题

#### 预览画布黑屏 / 不显示

- 检查文件是否成功加载（队列中应显示缩略图）
- 尝试切换预览模式（并排 / 叠加）
- 重置缩放：`Ctrl+0`
- 检查显卡驱动（WebView2 / WebKit 需要 GPU 加速）

#### 字体显示异常

- Windows：确保系统已安装「Microsoft YaHei」或「Segoe UI」
- Linux：安装 Noto 字体包：`sudo apt install fonts-noto`
- 在「设置 → 外观」中调整界面字体

### 性能问题

#### 应用卡顿 / 内存占用高

1. **减少并发数**：设置 → 队列 → 并发数改为 1
2. **关闭实时预览**：处理大文件时暂停实时预览
3. **限制输入边长**：参数面板 → 预处理 → 最大输入边长设为 1600-2400
4. **清理历史**：历史面板 → 清空历史（保留最近 20 条即可）

#### 启动速度慢

- 首次启动需要初始化 Python 环境，后续启动会更快
- 关闭「启动时检查更新」可略微提升启动速度
- 在 SSD 上运行应用，避免机械硬盘
